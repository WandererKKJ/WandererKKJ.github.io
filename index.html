pull request means merging codes to upstream remote storage with others too # git pull [원격저장소 이름] [원격저장소에서 pull하고자 하는 브랜치 이름] // 해당 브랜치의 파일들 로컬로 가져오기 # git branch insert // insert branch 생성 # git switch insert // insert branch 이동 # git branch // branch 확인 Edit your codes~~~~ # git add . // branch 디렉토리 로컬 git에 add # git commit -m '로그내용' // svn과 달리 로컬저장소에 반영 # git push [원격저장소 이름] [로컬저장소에서 push하고자 하는 브랜치 이름] then create pull request on github (rule로 되어있으면 무조건 해야겠지만, 그건 자유) after reviewing, merge pull request 여기서 핵심은 각자 contributor들의 원격저장소가 있다는 것인데, pull request를 요청함으로서 해당 원격저장소에서 upstream 원격저장소 즉, 모두가 관리하는 저장소로의 병합을 요청하는 것이다. (추가로 다른 contributor들의 저장소로도 pull하라는 request도 되는 것) 그래서 push request가 아닌 pull request인 것이, 다른 저장소로 하여금 내 변경사항을 검토하고 가져가라라는 뜻이기에.
